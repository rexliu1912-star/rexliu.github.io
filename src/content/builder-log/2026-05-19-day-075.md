<!-- en -->

- Connected the BIT.com read-only trade path into the portfolio workflow so executions can move from brokerage data into Convex and then into the public build chain.
- Tightened the Convex `recordTrade` path so closing a position can automatically clean up related monitor events, rules, and searches.
- Removed the earnings transcript step from the main portfolio pipeline, keeping that pipeline focused on health, scoring, snapshots, and source-of-truth sync.
- Completed and re-filed the MU / AAOI research package into the correct investment strategy topic directories, then re-ran scanner and quant validation.
- Tested Browserbase browse, then deliberately removed the CLI, shims, and skills because there was no stable account/API-key path or recurring use case yet.
- Re-verified the Xifeng mentor fallback path and the private Xifeng Compass UI, keeping it private rather than prematurely promoting it into the main product surface.
- The lesson today was restraint: a tool that installs successfully is still not worth keeping unless it earns a place in the operating loop.

<!-- zh -->

- 把 BIT.com 只读成交链路接进 Portfolio workflow，让成交能从券商数据进入 Convex，再进入公开页面构建链路。
- 收紧 Convex `recordTrade` 路径，清仓时自动清理相关 monitor events、rules 和 searches。
- 从主 Portfolio Pipeline 里移除 earnings transcript step，让它只负责组合健康度、评分、快照和事实源同步。
- 完成 MU / AAOI 研究包，并重新归档到正确的 investment strategy 主题目录，再重跑 scanner 和 quant 验证。
- 测试 Browserbase browse 后选择删除 CLI、shims 和 skills，因为还没有稳定账号/API key 路径，也没有高频使用场景。
- 重新验证 xifeng-mentor fallback 和 Xifeng Compass 私有 UI，但继续保持私有，不提前推成主产品入口。
- 今天的教训是克制：一个工具安装成功，不代表它值得留在系统里；除非它能进入真实运行闭环。
