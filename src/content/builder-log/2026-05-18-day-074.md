- Reworked the portfolio allocation logic so factual subgroups come from actual holdings instead of static display settings.
- Tightened the boundary between strategy and reality: targets can live in configuration, but current exposure has to come from the source of truth.
- Extended the enterprise software research set with ownership, fund-flow, and market-sentiment context so watchlist decisions are less narrative-only.
- Fixed a watchlist classification edge case where a research section could be mistaken for portfolio ownership.
- Ran a real video-to-knowledge workflow and turned a YouTube source into a durable transcript for later synthesis.
- The bigger lesson today was simple: a beautiful dashboard is still wrong if it keeps showing exposure that no longer exists.

<!-- zh -->

- 重做了 Portfolio Allocation 逻辑，让事实子分类从真实持仓生成，而不是从静态展示配置里读。
- 把策略和现实的边界收紧：目标配比可以留在配置里，但当前暴露必须来自事实源。
- 给 enterprise software 研报补上机构持仓、资金流和市场情绪，让 watchlist 判断不只是叙事。
- 修掉了 watchlist 分类里的一个边界问题，避免研究章节被误判成真实持仓。
- 跑通了一次视频到知识库的真实流程，把 YouTube 来源沉淀成后续可复用的 transcript。
- 今天最大的提醒很朴素：仪表盘再漂亮，如果还展示已经不存在的暴露，那就是错的。
