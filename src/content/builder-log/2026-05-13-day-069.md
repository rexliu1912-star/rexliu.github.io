<!-- en -->

- Simplified the portfolio watchlist so it speaks one scoring language instead of asking readers to reconcile competing signals.
- Reworked the factor display into a calmer visual system that feels like part of the site, not a dashboard bolted on later.
- Tightened the reading and podcast delivery path so long-form outputs go where they are useful and summaries stay lightweight.
- Fixed a podcast pipeline blind spot where discovery could be mistaken for completion, then added a retry loop for unfinished work.
- Retired an old intelligence entry point that had started overlapping with better-maintained daily workflows.
- Added a default supply-chain security rhythm for the local agent stack, with a clean baseline to compare against later.
- The big lesson was simple: a system improves when it has fewer places to look and clearer proof that the job actually finished.

<!-- zh -->

- Portfolio watchlist 收敛成一套评分语言，不再让读者在几套信号之间自己猜该信哪个。
- 因子展示改成更克制的视觉系统，看起来像网站自己的一部分，而不是后来硬贴上去的 dashboard。
- reading 和 podcast 的投递路径被理顺：长内容去该沉淀的地方，私聊只保留轻量摘要。
- podcast pipeline 补上一个盲点：发现内容不等于处理完成，未完成的内容要能回到重试队列。
- 一个和主流程重叠的旧情报入口被撤掉，少一个入口，主工作流反而更清楚。
- 本地 agent 栈开始有默认的供应链安全节奏，先有干净 baseline，未来才知道异常长什么样。
- 今天最重要的判断很简单：系统变强，不是因为多了几个页面，而是因为少了几个歧义、补上了完成证明。
