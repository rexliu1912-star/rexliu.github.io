- Tightened the portfolio page so closed-position notes, public records, and source data tell the same story instead of drifting apart.
- Moved the watchlist into a registry-driven flow, then connected it to the public portfolio refresh so research changes can show up as interface changes.
- Turned a thematic market idea into a module inside the watchlist scan rather than creating another separate monitor to babysit.
- Reworked the Intel HTML through several passes until it stopped looking like a generic dashboard and started feeling like a rexliu.io editorial issue.
- Promoted the investment research workflow into a stricter funnel, with report analysis and roundtable review as required stages instead of optional polish.
- Added missing specialist skills for research reports and roundtables, which makes the next single-name study less dependent on memory and more dependent on process.
- Refreshed the current AI and energy watchlists so the outputs are not just readable reports, but inputs for future scan rules and action triggers.
- The personal lesson today: a good report is still a dead object until it changes what the system does tomorrow.
