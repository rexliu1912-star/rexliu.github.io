<!-- en -->

- Built out the private mentor knowledge workflow, then deliberately pulled it back into a skill instead of letting it become another interface too early.
- Tightened portfolio presentation data so public pages can derive from the source of truth instead of relying on scattered display files.
- Added a market-data layer with caching, fallback behavior, and budget awareness, which makes the research pipeline feel less like scraping and more like infrastructure.
- Cleaned up community briefing delivery rules so mentions, invite-like links, and unsafe formatting get caught before publishing.
- Checked a dormant voice-extraction project honestly and marked the gap instead of pretending a backend existed.
- I liked the shape of today: less “ship a shiny surface,” more “decide where this capability belongs.” That is slower, but it compounds better.

<!-- zh -->

- 今天把私有导师语料工作流做通后，又主动收回到 skill 里，没有让它过早膨胀成另一个界面。
- 组合展示数据继续往事实源靠拢，公开页面应该从真实数据派生，而不是靠分散展示文件互相记住。
- 市场数据层补上了缓存、fallback 和预算意识，投研管线开始更像基础设施，而不是临时抓取。
- 社区日报的发布边界继续收紧，提及、邀请式链接和危险格式都要在发出前被机器拦住。
- 一个个人声音提取项目被诚实检查后标注了缺口：没有后端就说没有，不用假装已经能跑。
- 我喜欢今天的形状：不是急着多发一个漂亮入口，而是判断能力到底应该待在哪一层。慢一点，但更有复利。
