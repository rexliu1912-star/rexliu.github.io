## The Setup

This piece argues that the real shift in agentic commerce is not just new payment rails, but a new kind of business. Protocols like MPP and x402 let agents discover a service, pay for it, and receive output in one request. That sounds incremental until you look at what it removes: checkout pages, user accounts, subscription plans, customer-success overhead, and sometimes even the website itself.

The article’s key example is a machine-first marketplace where agents can already buy search across SEC filings, CAPTCHA solving, physical mail delivery, or image generation, often for fractions of a cent. In that world, the merchant is no longer a storefront. It is an endpoint with machine-readable docs and a price per call.

## Key Takeaways

- The buyer is changing. An agent arrives with a task, budget, and constraints already defined.
- Payment becomes authentication. If the agent can pay, it can use the service.
- Distribution shifts from marketing to discoverability plus reliability. Your API docs, pricing clarity, and uptime become the product.
- Per-request monetization reopens markets that subscriptions made uneconomic.

## Why It Matters

For Rex’s lens, this is bigger than a payments story. It suggests that agent infrastructure is starting to compress the go-to-market stack. If agents become meaningful demand sources, then many software businesses can be rebuilt as thin, high-margin service endpoints instead of full SaaS companies. That lowers the cost to launch, reduces human sales friction, and makes stablecoin or hybrid payment rails strategically important.

The deeper implication is that commerce may move from places to moments. A human used to browse, compare, register, and checkout. An agent simply detects a missing capability and transacts instantly. That favors businesses with the best machine interface, not the best brand theater.

## What to Watch

- Whether MPP, x402, or similar standards become default rails for agent purchases
- Whether major agent platforms expose native merchant discovery, not just payment
- Which categories break out first: data, scraping, compute, logistics, or compliance
- Whether SaaS vendors unbundle subscriptions into usage-priced agent endpoints

## 背景

这篇文章真正想讲的，不只是 Agent 支付通道成熟了，而是商业形态本身在变。像 MPP、x402 这类协议，让 Agent 可以在一次请求里完成发现服务、付款、获取结果三个动作。表面上看只是支付更顺滑，实质上它拿掉的是一整套旧互联网商业基础设施：结账页、注册账号、订阅套餐、客服体系，甚至官网本身。

文中举的案例已经很具体。一个面向机器的 marketplace 里，Agent 可以直接购买 SEC 文件全文搜索、验证码解决、实体信件寄送、图片生成等能力，价格低到千分之几美元一笔。在这种模式下，商家不再是“店铺”，而是一组接口、可读文档和按次计费规则。

## 关键要点

- 买方变了。以前是人来浏览和比较，现在是 Agent 带着明确任务、预算和约束直接进场。
- 支付本身就是认证。Agent 只要能完成付款，就能调用服务，不再需要传统账号体系。
- 分发逻辑也变了。决定你能不能成交的，不是首页设计，而是 API 文档是否清楚、价格是否透明、服务是否稳定。
- 按次收费重新打开了一批过去不成立的小生意。很多单次价值很低的能力，现在第一次具备了可持续变现条件。

## 为什么重要

如果站在 Rex 的视角，这不是单纯的支付赛道更新，而是 Agent 正在压缩整个 go-to-market stack。未来很多软件业务，也许不需要先长成一个完整 SaaS，再去配销售、漏斗、续费体系，而是可以先以“高毛利服务端点”的形式存在。谁先把能力产品化成机器可调用、机器可付费、机器可发现的接口，谁就更有机会吃到第一波 Agent 流量。

更深的一层是，商业正在从“场所”迁移到“时刻”。过去交易发生在网站、App、商店这些固定入口里。以后交易发生在 Agent 突然发现自己缺一项能力的那一刻。那时它不会看你的品牌故事，只会比较谁的接口最好用、最可靠、最便宜。

## 值得关注

- MPP、x402 这类协议，能否成为 Agent 采购的默认支付层
- 主流 Agent 平台会不会开放原生 merchant discovery，而不仅是支付能力
- 最先跑出来的是哪类“无头商家”：数据、抓取、算力、物流，还是合规服务
- 现有 SaaS 公司会不会把订阅产品拆成按次计费的 Agent 端点
