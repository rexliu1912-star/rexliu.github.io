## The Setup

In 1997, Tim Berners-Lee's team invented HTTP status code **402: Payment Required** — but left it unimplemented. There was no way to pay digitally. So the web took off anyway, and Google invented a workaround: **ads**.

The ad model worked because humans reading webpages can be distracted. That distraction became a $500B+ industry. But the model had one fatal assumption baked in: the user is human.

## The Argument

LLMs and agents don't get distracted. They don't notice banner ads. They don't pause to consider sponsored results. The entire economic architecture of the internet — from Google Search to Meta's feeds to affiliate links — was built on **monetizing human attention**. Agents have no attention to monetize.

The data is already brutal:
- Stack Overflow traffic: **down 75%** since GPT-4
- Tech news traffic: **down 60%**
- Computer-use agents now perfectly mimic real human browsing — breaking even walled garden defenses

The author draws a sharp analogy: **ChatGPT Checkout is AOL. Open Agentic Commerce is HTTP.**

AOL in 1998 was polished, curated, and doomed. HTTP was ugly, chaotic, and won. The same will happen with agentic commerce: OpenAI and Google's curated merchant catalogs require months of BD negotiations. Open protocols (HTTP for payments) let any server transact with any agent directly.

The 28-year-old 402 status code may finally get its implementation — via stablecoins and micropayment protocols built for agents.

## Why It Matters

This is one of the most important structural shifts Rex tracks: **the convergence of AI Agent infrastructure and crypto payment rails**.

If open protocols win (and history says they will), the winners are:
- **Stablecoin infrastructure** (USDC, payment rails) — the default settlement layer for agent-to-agent commerce
- **Open protocol builders** — whoever builds the HTTP equivalent for agentic payments
- **Platforms that expose APIs without gatekeeping** — early movers get compounding distribution

The losers: ad-dependent media, walled garden commerce, any business model that relies on human distraction.

**What to watch:**
- Which protocol emerges as the "HTTP" for agentic commerce (ACP, UCP, or something new)
- Stablecoin volume on Base/Ethereum as agent payment volume grows
- Circle's CPN (Corporate Payment Network) positioning — they spotted this thesis early

## 背景

1997 年，Tim Berners-Lee 团队发明了 HTTP 状态码 **402：Payment Required**——但从未真正实现它。当时没有可靠的数字支付方式。于是互联网就这样运转了，Google 发明了一个替代方案：**广告**。

广告模式之所以奏效，是因为浏览网页的人类可以被分心。这种"分心"催生了一个 5000 亿美元以上的行业。但这个模式有一个致命的隐含假设：用户是人类。

## 核心论点

LLM 和 AI Agent 不会被分心。它们看不到横幅广告，不会停下来考虑赞助结果。互联网整个经济架构——从 Google 搜索到 Meta Feed 到联盟营销——都建立在**变现人类注意力**的基础上。Agent 没有注意力可以变现。

数据已经很残酷：
- Stack Overflow 流量：GPT-4 发布后**下降 75%**
- 科技新闻流量：**下降 60%**
- 计算机操控型 Agent 现在能完美模拟真实人类浏览行为——连封闭花园的防御也被突破了

文章的核心类比非常犀利：**ChatGPT Checkout 是 AOL，开放 Agent 商业是 HTTP。**

1998 年的 AOL 精心策划、外表光鲜、注定失败。HTTP 丑陋、混乱、最终胜出。Agent 商业也会上演同样的剧情：OpenAI 和 Google 的精选商户目录需要数月的 BD 谈判，而开放协议（支付领域的 HTTP）能让任意服务器直接与任意 Agent 交易。

那个沉睡了 28 年的 402 状态码，可能终于要通过稳定币和专为 Agent 构建的微支付协议找到它的实现了。

## 为什么重要

这是 Rex 持续追踪的最重要的结构性转变之一：**AI Agent 基础设施与加密支付轨道的融合**。

如果开放协议胜出（历史说会），赢家是：
- **稳定币基础设施**（USDC、支付轨道）——Agent 间商业的默认结算层
- **开放协议构建者**——谁建出 Agent 支付领域的 HTTP 谁就赢
- **无门槛开放 API 的平台**——先行者获得复利式分发优势

输家：依赖广告的媒体、封闭花园商业、任何依赖人类注意力的商业模式。

**值得关注：**
- 哪个协议成为 Agent 商业领域的"HTTP"（ACP、UCP 还是新东西）
- Base/以太坊上的稳定币交易量——随着 Agent 支付量增长
- Circle 的 CPN（企业支付网络）定位——他们很早就看到了这个逻辑
